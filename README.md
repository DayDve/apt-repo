Personal APT repository for software that isn't in standard Ubuntu/Debian repos — either too niche, too new, or requires special build machinery. Every package is rebuilt from source (or repackaged) by CI on every upstream update and published as a `.deb`.

## Available packages

| App | Description | Source | Build method |
|---|---|---|---|
| **ayugram** | ToS-breaking Telegram client with enhanced features | [AyuGram/AyuGramDesktop](https://github.com/AyuGram/AyuGramDesktop) | CentOS Docker build env, cmake, ccache |
| **bees** | Best-Effort Extent-Same, a btrfs deduplication daemon | [Zygo/bees](https://github.com/Zygo/bees) | Ubuntu, devscripts, dpkg-buildpackage |
| **grub-btrfs** | GRUB menu entries for btrfs snapshots | [Antynea/grub-btrfs](https://github.com/Antynea/grub-btrfs) | Ubuntu, make install, dpkg-deb |
| **keyd** | Key remapping daemon | [rvaiya/keyd](https://github.com/rvaiya/keyd) | Ubuntu, make, dpkg-deb |
| **rdm** | Remote Desktop Manager (prebuilt .deb) | [Devolutions](https://devolutions.net/remote-desktop-manager/) | Alpine download pass-through |

## Install

```bash
# Add the repository key
sudo mkdir -p /etc/apt/keyrings
sudo curl -fsSL https://daydve.github.io/apt-repo/apt-key.asc \
  -o /etc/apt/keyrings/daydve-apt-repo.asc

# Add the repository (replace noble with your distro codename)
echo "deb [signed-by=/etc/apt/keyrings/daydve-apt-repo.asc] https://daydve.github.io/apt-repo $(lsb_release -cs) main" \
  | sudo tee /etc/apt/sources.list.d/daydve-apt-repo.list

sudo apt update
```

Install a package:

```bash
sudo apt install bees
```

## For maintainers

### Structure

```
apps/
├── build.sh               # build any app locally
├── apt-key.asc            # public GPG key (auto-generated by CI)
└── <app>/
    ├── Dockerfile          # multi-stage build → produces .deb at /
    ├── check_update        # exit 0 if upstream has a newer version
    └── get_version         # stdout: version=... \n --- \n changelog
```

### Build locally

```bash
bash apps/build.sh <app>           # default distro: noble
bash apps/build.sh <app> <ver>     # explicit version
bash apps/build.sh <app> <ver> oracular  # different distro
```

Requires Docker with BuildKit. On CI, GHA cache (`type=gha`) is used; local builds always run from scratch.

### CI

| Workflow | Trigger | What it does |
|---|---|---|
| **check-updates.yml** | Every 6h + manual (`?app=<name>`) | Checks all apps for upstream updates, builds changed ones, triggers deploy |
| **deploy-apt.yml** | Triggered by check-updates or manually | Rebuilds `dists/` + `pool/` on the `apt` branch with signed Release metadata |

Manual update: go to Actions → Check updates → "Run workflow" → optionally set app name.

### Release tags

Named `<app>-<version>` (e.g. `bees-0.10-1-gb1c2a3e`). Created by CI on main/master pushes. `.deb` assets are attached to each release and later swept into `pool/` by `deploy-apt.yml`.

### Versioning

Each app's `get_version` script determines the current upstream version. For GitHub-hosted apps, the version is `<latest_tag>-<commits_ahead>-g<sha>`. For prebuilt software like RDM, the version is scraped from the vendor's download page.
